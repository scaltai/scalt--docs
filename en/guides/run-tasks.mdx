## Introduction

## Immediate execution

### Launch a task manually

<Steps>
  <Step title="Select your agent">
    In your project, click on the agent to execute
  </Step>
  <Step title="Click Execute">
    **"Execute now"** button in top right
  </Step>
  <Step title="Provide parameters">
    If agent requires input data, fill out the form
  </Step>
  <Step title="Launch execution">
    Confirm to start the task
  </Step>
</Steps>

### Custom scheduling (Cron)

For specific needs, use cron syntax:

<CodeGroup>

```cron Cron syntax examples
# Format: minute hour day month day_of_week

# Every weekday at 9:00 AM
0 9 * * 1-5

# Every Monday and Wednesday at 2:30 PM
30 14 * * 1,3

# 1st and 15th of each month at 8:00 AM
0 8 1,15 * *

# Every 15 minutes
*/15 * * * *

# Every day at 9am, 12pm and 6pm
0 9,12,18 * * *

# Last day of each month at midnight
0 0 28-31 * *
```

```yaml Complete configuration
schedule:
  name: "Weekly report"
  cron: "0 9 * * 1"  # Every Monday at 9am
  timezone: "Europe/Paris"
  enabled: true
  
  # Optional parameters
  params:
    date_range: "last_7_days"
    format: "pdf"
  
  # Notifications
  notifications:
    on_start:
      - type: "slack"
        channel: "#notifications"
        message: "üöÄ Generating weekly report..."
    
    on_success:
      - type: "email"
        to: ["team@company.com"]
        subject: "Weekly report available"
        attach_output: true
    
    on_failure:
      - type: "slack"
        channel: "#alerts"
        message: "‚ùå Report generation failed"
        mention: ["@admin"]
```

</CodeGroup>

<Tip>

## Notifications

Stay informed about your agent executions.

### Configure notifications

<Tabs>
  <Tab title="Email">
    ```yaml
    notifications:
      email:
        enabled: true
        recipients: ["admin@company.com", "team@company.com"]
        events:
          - on_failure
          - on_success_if_critical
        include_logs: true
        include_results: true
    ```
  </Tab>
  
  <Tab title="Slack">
    ```yaml
    notifications:
      slack:
        enabled: true
        webhook_url: "${secrets.SLACK_WEBHOOK}"
        channel: "#notifications"
        events:
          - on_start
          - on_success
          - on_failure
        mentions:
          on_failure: ["@admin", "@devops"]
    ```
  </Tab>
  
  <Tab title="Microsoft Teams">
    ```yaml
    notifications:
      teams:
        enabled: true
        webhook_url: "${secrets.TEAMS_WEBHOOK}"
        events:
          - on_failure
        adaptive_card: true
    ```
  </Tab>
</Tabs>

## Best practices

<AccordionGroup>
  <Accordion icon="lightbulb" title="Scheduling">
    - Avoid peak hours for heavy tasks
    - Space out executions to avoid overloading third-party APIs
    - Use explicit timezones
    - First test with low frequency
  </Accordion>
  
  <Accordion icon="shield-check" title="Reliability">
    - Always configure a retry strategy
    - Enable failure notifications
    - Regularly monitor history
    - Keep a fallback agent
  </Accordion>
  
  <Accordion icon="dollar-sign" title="Costs">
    - Use economical models for simple tasks
    - Limit number of parallel executions
    - Archive obsolete agents
    - Monitor credit consumption
  </Accordion>
</AccordionGroup>

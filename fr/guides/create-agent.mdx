## Introduction

L'**Assistant Scalt** est votre copilote pour créer des agents intelligents. Plutôt que de configurer manuellement chaque paramètre, vous décrivez simplement ce que vous voulez automatiser, et l'Assistant génère l'agent pour vous.

<Info>
  **Simple comme une conversation. Puissant comme du code.**
</Info>

## Comment fonctionne l'Assistant Scalt ?

L'Assistant Scalt utilise l'intelligence artificielle pour :

1. **Comprendre** votre besoin exprimé en langage naturel
2. **Analyser** les outils et intégrations disponibles
3. **Générer** la configuration optimale de l'agent
4. **Suggérer** des améliorations et optimisations

## Accéder à l'Assistant Scalt

<Steps>
  <Step title="Ouvrez votre projet">
    Sélectionnez le projet dans lequel vous souhaitez créer l'agent
  </Step>
  <Step title="Lancez l'Assistant">
   Sélectionnez **"Assistant Scalt"**
  </Step>
  <Step title="Commencez la conversation">
    L'interface de l'Assistant s'ouvre, prête à recevoir vos instructions
  </Step>
</Steps>

## Écrire un bon prompt

### Principes de base

Un bon prompt pour l'Assistant Scalt doit être :

<CardGroup cols={2}>
  <Card title="Clair" icon="lightbulb">
    Exprimez clairement l'objectif de l'agent
  </Card>
  <Card title="Spécifique" icon="bullseye">
    Précisez les actions à effectuer
  </Card>
  <Card title="Contextuel" icon="book">
    Donnez le contexte métier nécessaire
  </Card>
  <Card title="Complet" icon="list-check">
    Incluez les déclencheurs et résultats attendus
  </Card>
</CardGroup>

### Structure recommandée

Utilisez cette structure pour de meilleurs résultats :

```text
[OBJECTIF] : Que doit faire l'agent ?
[DÉCLENCHEUR] : Quand doit-il agir ?
[ACTIONS] : Quelles étapes doit-il suivre ?
[RÉSULTAT] : Quel est le livrable attendu ?
```

## Exemples de prompts

### Exemple 1 : Agent de support client

<CodeGroup>

```text Prompt basique
Créer un agent qui répond aux emails de support client
```

```text Prompt optimisé
OBJECTIF : Automatiser les réponses aux demandes de support client

DÉCLENCHEUR : Chaque fois qu'un email arrive sur support@entreprise.com

ACTIONS :
1. Analyser le contenu de l'email
2. Identifier le type de demande (technique, facturation, autre)
3. Chercher dans notre base de connaissances
4. Générer une réponse appropriée
5. Si la demande est complexe, créer un ticket et alerter l'équipe

RÉSULTAT : 
- Email de réponse envoyé au client
- Ticket créé dans Zendesk si nécessaire
- Notification Slack pour les cas urgents
```

</CodeGroup>

<Tip>
  Plus votre prompt est détaillé, plus l'agent généré sera précis et efficace.
</Tip>

## Affiner votre agent avec l'Assistant

L'Assistant Scalt est conversationnel. Vous pouvez affiner votre agent en plusieurs échanges :

<Steps>
  <Step title="Prompt initial">
    Décrivez votre besoin général
  </Step>
  <Step title="L'Assistant propose">
    Il génère une première version de l'agent
  </Step>
  <Step title="Vous affinez">
    "Ajoute aussi une vérification anti-spam"
    "Utilise le modèle GPT-4 au lieu de Claude"
    "Envoie les notifications sur Teams plutôt que Slack"
  </Step>
  <Step title="Itération">
    L'Assistant ajuste la configuration
  </Step>
  <Step title="Validation">
    Une fois satisfait, validez la création
  </Step>
</Steps>

## Tester votre agent

Une fois créé, testez immédiatement votre agent :

<Steps>
  <Step title="Accédez au Playground">
    L'Assistant vous redirige automatiquement vers le Playground
  </Step>
  <Step title="Simulez des scénarios">
    Testez avec des données réelles ou des exemples
  </Step>
  <Step title="Vérifiez les résultats">
    Examinez les actions effectuées et les résultats
  </Step>
  <Step title="Ajustez si nécessaire">
    Retournez à l'Assistant pour des modifications
  </Step>
</Steps>

## Limites et considérations

<Warning>
  **Points d'attention** :
  
  - L'Assistant génère une configuration de base. Pour des besoins très complexes, une édition manuelle peut être nécessaire.
  - Vérifiez toujours les permissions et accès aux outils tiers.
  - Testez en profondeur avant de déployer en production.
  - Les agents génèrent des coûts (crédits Scalt + APIs tierces).
</Warning>
